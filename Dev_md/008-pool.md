## 직접 구현한 2D 물리 엔진

외부 라이브러리(matter.js 등) 없이 ~250줄 JS만으로 당구 물리를 처리한다.

### 1) 공-공 탄성 충돌
질량이 같다고 가정하고, 충돌축(법선) 방향 속도 성분만 교환:

```js
const dx = b.x - a.x, dy = b.y - a.y;
const dist = Math.hypot(dx, dy);
if (dist < BALL_R * 2) {
  const nx = dx / dist, ny = dy / dist;
  // 겹침 해소
  const overlap = (BALL_R * 2 - dist) / 2;
  a.x -= nx * overlap; a.y -= ny * overlap;
  b.x += nx * overlap; b.y += ny * overlap;
  // 상대 속도의 충돌축 성분만 교환
  const dvx = b.vx - a.vx, dvy = b.vy - a.vy;
  const dot = dvx * nx + dvy * ny;
  if (dot < 0) {
    a.vx += dot * nx; a.vy += dot * ny;
    b.vx -= dot * nx; b.vy -= dot * ny;
  }
}
```

### 2) 마찰
매 프레임 속도에 0.985를 곱하고, 0.05 이하면 0으로 스냅. 자연스러운 감속 + 미세 떨림 제거.

### 3) 벽 반사
벽에 닿으면 반대 방향 속도로 반전 + 90% 감쇠. 위치도 벽 안쪽으로 보정.

### 4) 포켓 검출
6개 포켓과의 거리가 `POCKET_R - 4` 미만이면 공이 빠진 것으로 처리. 큐볼이 빠지면 "스크래치"로 처리하고 큐볼 위치를 헤드 라인으로 복귀시킨다.

## 시각 디자인

- 펠트(녹색 천): radial gradient + inset shadow로 둥근 표면감
- 큐대: 갈색 그라데이션 + 짧은 팁(연한 갈색) + 긴 손잡이(진한 갈색)
- 공: 그림자 + 본체 + 하이라이트 3겹. 줄무늬 공(9~15번)은 위/아래 호 두 개로 표현
- 가이드 라인: dash + 40% 투명 흰색

## 조준 UX

1. 큐볼 위에서 마우스 다운
2. 드래그하면 거리에 비례해 power 증가 (max 18)
3. 마우스 업 → 발사 (드래그 반대 방향)
4. 큐 스틱이 파워에 따라 뒤로 당겨지는 애니메이션

같은 흐름이 터치 이벤트에도 동일하게 동작.

## 게임 규칙 (단순화)

원래 8-ball 룰은 솔리드/스트라이프 색깔 그룹을 결정하고 8번 공은 마지막에 처리하는 등 복잡해서, 한 페이지 샘플 답게 단순화했다:
- 2인 교대 (P1, P2)
- 어느 색이든 포켓에 넣으면 해당 차례 플레이어가 +1점
- **모든 샷 후 차례 넘김** (정통은 점수 내면 한 번 더지만 단순화)
- 큐볼이 빠지면 차례 즉시 넘김 + 큐볼 복귀
- 먼저 5점 → 승리
