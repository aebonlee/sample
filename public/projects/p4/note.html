다음 릴레이션이 BCNF를 만족하지 않는 이유로 가장 적절한 것은?

R(학번, 과목번호, 강사번호)
함수 종속:
  - {학번, 과목번호} → 강사번호
  - 강사번호 → 과목번호
① 모든 속성이 원자값이 아니어서
③ 결정자가 후보키가 아닌 함수 종속이 존재
💡 BCNF는 "모든 결정자가 후보키여야 한다"는 조건. 위에서 강사번호 → 과목번호는 결정자(강사번호)가 후보키가 아니므로 BCNF를 위반.

다음 C 코드의 출력 결과는?

int arr[] = {1, 2, 3, 4, 5};
int *p = arr;
printf("%d", *(p+2) + *(p+3));
① 5
③ 7
💡 포인터 산술: *(p+2) = arr[2] = 3, *(p+3) = arr[3] = 4. 따라서 3 + 4 = 7.
